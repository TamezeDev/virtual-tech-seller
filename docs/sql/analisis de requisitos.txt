1. Requisitos funcionales:

- Se efectuarán exposiciones de ventas de productos.
- Habrá 3 categorías de usuarios: Administradores, Moderadores y Clientes
- Los productos pueden ser nuevos o usados.
- Los productos pertenecen a categorías.
- Los productos usados tendrán descuento según el estado en que se encuentren
- Los administrados son los encargados de subir/eliminar artículos de las exposiciones
- Los administradores registran a los clientes, consultores u otros administradores
- Los administradores pueden importar / exportar productos en formato XML
- Los clientes pueden registrarse pero deben esperar autorización de algún administrador
- Los clientes pueden acceder a las exposiciones
- Los moderadores pueden consultar y filtrar como van las ventas de cada exposición
- Los moderadores pueden consultar y filtrar las visitas a las exposiciones
- Los clientes pueden ojear y filtrar para ver el catálogo de la exposición
- Los clientes compran productos
- Los clientes pueden ver la lista de sus productos comprados
- Los clientes pueden recargar crédito para seguir comprando

- Se deben registrar las ventas para saber todo lo que se vende en cada exposición
- Se deben registrar las visitas a eventos para contabilizarlas los moderadores

- FUNCIONES EXTRA DE MPO:
a) Los clientes tendrán un carrito de compra. Al meter un artículo, en caso que otro cliente se adelante a la compra o un administrador lo quite de un evento, este artículo no podrá comprarlo ya el cliente y se actualizara su carrito al stock disponible.
b) Las contraseñas deben estar protegidas para que no sean visibles en la base de datos mediante una encriptación
c) Los administradores tienen el control y pueden modificar datos importantes como eventos o productos pero con limitaciones:
 - Un evento que ya está iniciado o finalizado tiene restringido el cambio de datos. Se puede modificar solo los que aún no han comenzado
 - Un producto registrado puede cambiar de precio y añadirle mas stock(en caso de ser nuevo). El nombre y sus características no se podrán cambiar


2. Requisitos no funcionales.

- Se requiere de un diseño acorde al entorno tecnológico
- El cliente pide una portada atractiva y llamativa
- La interfaz debe estar pensada para equipos de escritorios o laptops
- Los productos tendrán un nombre y su descripción con una imagen acorde, en caso de no disponer de imagen se pondrá una genérica
- Los menús con botones fáciles de acceder y de carga rápida entre navegación


2. Requisitos de diseño en la APP

- Debe haber uso coherente de herencia y polimorfismo (con interfaces que detallen el qué puede hacer un objeto si se necesitan)
- Debe  haber una función que exporte a XML los artículos y estén validados con un XSD o bien que para meter nuevo artículos se genere antes un XML y el XSD lo valide antes de subirlo a la base de datos
- Se debe de añadir una funcionalidad extra de alta importancia
- Se debe estructurar todo por paquetes de forma organizada
- Cada clase debe tener una única responsabilidad así como por ejemplo los controladores gestionan las vistas o los servicios acuden a los repositorios para obtener los datos del servidor y proporcionarlos al controlador.
- Detallar en el informe las dependencias usadas
